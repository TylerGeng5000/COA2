# 最小指令集（基线 + 完整表）

本页依据课设文档（Microprogrammed CPU Design）整理：

- 存储器：256×16
- 指令字：16-bit
- 格式：`[15:8]=opcode`，`[7:0]=addr/imm`
- 数据：二进制补码表示

> 目标：先用“基线指令集”跑通示例程序（1..100求和），再补齐“完整指令集”。

## 1. 基线指令集（建议先实现）

这些指令足以跑通课设给的求和示例：

| 指令 | opcode(二进制) | 语义（伪代码） |
|---|---|---|
| `STORE X` | `0000_0001` | `MEM[X] <- ACC` |
| `LOAD X` | `0000_0010` | `ACC <- MEM[X]` |
| `ADD X` | `0000_0011` | `ACC <- ACC + MEM[X]` |
| `SUB X` | `0000_0100` | `ACC <- ACC - MEM[X]` |
| `JMPGEZ X` | `0000_0101` | `if ACC>=0 then PC<-X else PC<-PC+1` |
| `JMP X` | `0000_0110` | `PC <- X` |
| `HALT` | `0000_0111` | 停机（冻结状态） |

### 分支条件约定

- `ACC>=0` 以 `ACC[15]==0` 判断（补码符号位）。

## 2. 完整指令集（按课设 Table 1）

| 指令 | opcode(二进制) | 语义（团队需统一细节） |
|---|---|---|
| `MPY X` | `0000_1000` | `ACC * MEM[X] -> MR, ACC` |
| `AND X` | `0000_1010` | `ACC <- ACC & MEM[X]` |
| `OR X` | `0000_1011` | `ACC <- ACC | MEM[X]` |
| `NOT X` | `0000_1100` | `ACC <- ~MEM[X]` |
| `SHIFTR` | `0000_1101` | `ACC <- MEM[X] >> 1`（逻辑右移） |
| `SHIFTL` | `0000_1110` | `ACC <- MEM[X] << 1`（逻辑左移） |

> 说明：课设文档中 `NOT/SHIFT` 的描述以 `[X]` 为操作数来源。为了实现一致性，建议**统一约定**：这些指令都先把 `MEM[X]` 读入 `BR`，再由 ALU 完成 `ACC` 写回。

> 复位与时序提示：课设明确“所有寄存器正沿触发、复位信号与时钟同步”。因此实现与测试时请统一使用**同步复位**（否则仿真与上板可能出现不一致）。

## 3. MPY 的结果拆分约定（建议）

课设只说“结果写入 MR, ACC”，未明确高/低字。

- 建议团队约定：
  - 32-bit 乘积 `P = signed(ACC) * signed(MEM[X])`
  - `ACC <- P[15:0]`（低 16 位）
  - `MR  <- P[31:16]`（高 16 位）

如果助教/老师有明确规定，以课堂规定为准；一旦修改必须同步更新 datapath 与测试用例。

## 4. 内存初始化文件约定（建议）

- 256 行，每行 16-bit HEX（例如 `02A0`）
- 地址从 `00` 开始顺序填充（便于仿真/上板一致）

## 5. 最小验收用例（建议）

- 用例1：课设给的 1..100 求和
- 用例2：分支：`ACC` 正/负两条路径都覆盖
- 用例3：读写一致性：`STORE` 后立刻 `LOAD`
- 用例4：`HALT` 行为稳定（停机后状态不变）
